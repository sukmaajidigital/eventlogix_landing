Arsitektur Sistem Premium

Dirancang untuk performa tinggi, disesuaikan untuk setiap jenis acara.

Empat modul inti yang berjalan di balik setiap undangan digital Anda — dari pengaturan domain instan hingga validasi check-in instan di lokasi acara.

Module 01 / Domain & Tenant

Multi-Tenant & Domain Kustom

Setiap event otomatis mendapat subdomain instan begitu dibuat. Ingin menggunakan domain ber-brand klien Anda? Cukup arahkan CNAME, dan sistem kami secara otomatis memasang SSL aman secara real-time.

CNAME Mapping • Let's Encrypt Auto-SSL • Isolated DB Tenant

Module 02 / Template Engine

Dynamic JSON Theming

Susun tata letak undangan dengan blok modular (Hero, Galeri, Kisah Acara, RSVP, Penunjuk Peta). Seluruh konfigurasi disimpan dalam format JSON terstruktur. Kustomisasi font, warna, dan urutan seksi dengan mudah.

Dynamic Layout Block • JSON Config Schema • Google Fonts API

Module 03 / RSVP Core

RSVP & Manajemen Tamu Fleksibel

Form RSVP adaptif. Anda dapat menambahkan field input kustom (jumlah tamu tambahan, pantangan makanan, preferensi sesi seminar) berdasarkan skema metadata dinamis (Entity-Attribute-Value).

EAV Schema • Custom Field Input • Instant Notification

Module 04 / Gateway & Check-in

Tautan Personal & QR Code

Tamu menerima tautan unik yang mempersonalisasi konten undangan di browser mereka. Dilengkapi QR Code bawaan untuk dipindai oleh panitia di lokasi guna verifikasi kehadiran secepat 0.8 detik.

Personalized URL Query • QR Code API • Mobile Scanner Portal

Tanya Jawab

Pertanyaan Yang Sering Diajukan

Setiap kali Anda membuat acara baru di eventlogix.id, sistem kami mengalokasikan entri tenant baru secara virtual. Hal ini memberikan subdomain unik (seperti namaevent.eventlogix.id) yang langsung mengarah ke konfigurasi JSON tersendiri yang mengisolasi data tamu dan form RSVP dari acara lainnya.
Ya. Cukup instruksikan klien Anda untuk menambahkan entri CNAME di manajemen DNS domain mereka (misal: mengarahkan undangan.nikah-alicebob.com ke CNAME tenant.eventlogix.id). Begitu diarahkan, sistem reverse-proxy kami akan mendeteksi permintaan baru dan memasang sertifikat SSL dari Let's Encrypt secara otomatis dalam beberapa menit.
Alih-alih menyusun kode HTML/CSS manual yang rumit untuk setiap acara, sistem kami mendefinisikan undangan dalam skema JSON. Skema ini menentukan komponen apa saja yang muncul (misalnya: Cover, Galeri Foto, Informasi Waktu, Form RSVP) beserta parameter warna dan font. Hal ini memungkinkan Anda untuk merancang ulang tata letak dalam hitungan detik tanpa risiko mematahkan fungsionalitas RSVP atau check-in.
Setiap tamu yang terdaftar di sistem akan mendapatkan tautan undangan personal yang unik. Di dalam undangan tersebut terdapat kode QR individual. Saat acara berlangsung, panitia EO dapat memindai kode QR tamu menggunakan portal scanner mobile kami dari browser HP mereka. Sistem akan secara real-time memverifikasi nama, mencatat status check-in, dan memvalidasi jika QR Code tersebut sudah pernah dipakai sebelumnya.
